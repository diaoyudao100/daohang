## 一、整体布局

```
┌─────────────────────────────────────────┐
│  [便签]  [编辑模式]  [切换主题]          │  ← 顶栏（右对齐，紧凑）
├─────────────────────────────────────────┤
│           04:32:15                      │  ← 大时钟（字体细、渐变色）
│        2026年3月17日  星期二             │  ← 日期行
│           下午好 👋                     │  ← 问候语
├─────────────────────────────────────────┤
│               🌙                        │
│            我的导航                     │  ← Hero区（头像+标题，紧凑）
│         收藏常用网站 · 一键直达          │
│  ┌──────────────────────────────────┐  │
│  │ Google Bing 百度 B站 GitHub      │  │  ← 搜索引擎切换胶囊
│  │ [搜索框              ] [🔍]      │  │  ← 搜索框
│  └──────────────────────────────────┘  │
├─────────────────────────────────────────┤
│  📅 3月17日  ⏰ 04:32:15  🗓 星期二  📆 第12周  │  ← 信息条
├─────────────────────────────────────────┤
│  🔧常用工具  💻开发资源  🤖AI工具  ...   │  ← Tab导航栏
├─────────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐  │
│  │ 卡 │ │ 卡 │ │ 卡 │ │ 卡 │ │ 卡 │  │  ← 书签卡片网格
│  └────┘ └────┘ └────┘ └────┘ └────┘  │
```

**布局原则：** 顶部区域（时钟+Hero+搜索+信息条）总高度尽量压缩，让书签卡片区域在首屏尽量多显示。

## 二、功能模块

### 2.1 实时时钟
- `setInterval` 每秒更新
- 显示：时:分:秒（大字渐变色）
- 显示：年月日 星期X
- 显示：问候语（按小时段：早上好/上午好/下午好/晚上好/深夜好）
- 信息条同步更新：日期、时间、星期、第几周

### 2.2 搜索
- 搜索引擎：Google / Bing / 百度 / B站 / GitHub（胶囊按钮，单选）
- 搜索框：Enter 或点按钮触发
- 自动识别：输入 `https://` 开头直接跳转，否则拼接搜索URL
- 新标签页打开

### 2.3 书签卡片
- 每张卡片：图标（Emoji）+ 名称 + 描述
- hover：上浮 + 描边高亮 + 图标放大
- 点击：新标签页打开
- 卡片网格：`auto-fill minmax(180px, 1fr)`，自适应列数

### 2.4 分类 Tab
- 默认6个分类：常用工具 / 开发资源 / AI工具 / 设计素材 / 娱乐休闲 / 学习阅读
- 点击切换，当前分类高亮下划线
- 编辑模式下可新增自定义分类

### 2.5 编辑模式
- 右上角按钮切换进入/退出
- 进入后：
  - Tab栏末尾出现「＋ 分类」按钮
  - 每个分类区域顶部出现「＋ 添加书签」按钮
  - 每张卡片悬停时出现「编辑」按钮
- 退出后自动保存到 localStorage
- 卡片点击在编辑模式下打开编辑弹窗（不跳转）

### 2.6 添加/编辑书签弹窗
- 字段：URL、名称、描述、图标（Emoji）、所属分类
- **AI自动补全**：输入URL后实时匹配，自动填充名称/描述/图标
  - 本地规则库覆盖40+常用网站
  - 未匹配时从域名提取名称
  - 提示行显示识别结果（`✨ AI识别：xxx`）
- 编辑模式下底部有「删除」按钮（红色，需确认）
- 点击遮罩层关闭

### 2.7 新增分类弹窗
- 字段：分类名称、图标（Emoji）
- 保存后自动切换到新分类

### 2.8 便签
- 右上角按钮打开
- 多行文本框，内容保存到 localStorage
- 保存/关闭按钮

### 2.9 明暗主题
- 右上角按钮切换
- 深色（默认）/ 浅色
- 偏好保存到 localStorage

### 2.10 粒子背景
- Canvas 全屏粒子动画（50个粒子，连线效果）
- fixed定位，不影响交互
- 主题切换时粒子颜色同步变化

## 五、技术方案

### 5.1 整体架构

```
┌─────────────────────────────────────────────┐
│            Cloudflare Pages                  │
│         （托管静态前端 index.html）           │
└──────────────────┬──────────────────────────┘
                   │ fetch API
┌──────────────────▼──────────────────────────┐
│          Cloudflare Workers                  │
│         （API 路由 + 密码校验）               │
└──────────────────┬──────────────────────────┘
                   │ KV read/write
┌──────────────────▼──────────────────────────┐
│           Cloudflare KV                      │
│  key: "nav_data" → { cats, cards }           │
│  key: "nav_pwd"  → 密码hash（SHA-256）        │
└─────────────────────────────────────────────┘
```

### 5.2 文件结构

```
daohang/
├── index.html          # 前端单文件
├── worker.js           # Cloudflare Worker 脚本
└── wrangler.toml       # Cloudflare 部署配置
```

### 5.3 Worker API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | `/api/data` | 获取所有书签和分类数据（公开） |
| POST | `/api/data` | 保存数据（需密码） |
| POST | `/api/verify` | 验证密码是否正确 |

**POST /api/data 请求体：**
```json
{
  "password": "用户输入的密码",
  "cats": [...],
  "cards": [...]
}
```

**密码校验方式：**
- 密码以 SHA-256 哈希存储在 KV（key: `nav_pwd`）
- 第一次部署时在 wrangler.toml 或 KV 中预设密码哈希
- Worker 收到请求后对比哈希，匹配才写入

### 5.4 前端同步逻辑

1. **页面加载**：`GET /api/data` 拉取远端数据，合并到本地（远端优先）
2. **编辑模式保存**：
   - 点击「完成编辑」时，若检测到数据有变更
   - 弹出密码输入框
   - 输入密码后 `POST /api/data` 同步到 KV
   - 成功提示「✅ 已同步到云端」，失败提示「❌ 密码错误」
3. **离线降级**：若 Worker 不可达，数据保存到 localStorage，不影响本地使用

### 5.5 前端技术
- **纯静态单文件** `index.html`，无构建工具依赖
- **无框架**，原生 JS + CSS 变量
- **数据优先级**：云端 KV > localStorage > 默认数据
- **构建方式**：Node.js 脚本生成（避免 shell heredoc 引号问题）
- 文件位置：`D:/claude/daohang/index.html`

## 七、已确认事项

### 前端
1. 头像区域保留，支持自定义（点击头像上传/输入图片URL）
2. 导航标题「我的导航」可编辑（编辑模式下点击标题修改）
3. 书签支持拖拽排序（同分类内拖拽，HTML5 drag API）
4. 支持书签导入/导出（JSON格式）
5. 搜索引擎列表暂不自定义（保持5个固定）

### 云同步
6. **首次访问设置密码**：KV 中无密码记录时，弹出初始化页面让用户设置密码
7. **多账户**：每个账户有独立用户名+密码，数据按账户隔离存储
   - KV key 结构：`user:{username}:data`、`user:{username}:pwd`
   - 登录后 token 存 sessionStorage，关闭标签页失效
8. **自动同步**：编辑有改动时，防抖 2 秒后自动同步到 KV（需已登录）
   - 未登录状态只存 localStorage
   - 同步中显示「同步中…」，成功显示「✅ 已同步」+时间，失败显示「❌ 同步失败」
9. **部署方案**：Worker 作为 Pages 的 Functions（同域），路径 `/api/*`
   - 前端和 Worker 同在一个 Cloudflare Pages 项目
   - 无跨域问题
10. **显示上次同步时间**：顶栏同步状态区显示「上次同步 HH:mm:ss」

### 账户功能
- 右上角显示登录状态：未登录显示「🔑 登录」，已登录显示「👤 用户名 ▾」
- 点击用户名下拉：切换账户 / 退出登录
- 支持注册新账户（首次无账户时引导注册）
- 密码在 Worker 端 SHA-256 哈希后存储，前端明文传输（依赖 HTTPS）
